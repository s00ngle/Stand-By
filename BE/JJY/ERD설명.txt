User table
단일기본키 userId
-설명-
유저에 대한 기본적인 정보를 담는다
기본키 값은 UUID를 통해 저장하는 것을 고려한다.

Profile table
단일기본키 userId
-설명-
User table의 기본키를 상속받는다.
유저의 자기소개에 대한 정보를 담는다.
한명의 유저는 하나의 프로필을 갖는다
유저가 삭제되면 프로필이 삭제된다

Resume Table
복합기본키 userId, resumeId 
-설명-
이력데이터를 담는 것이다.
한명의 유저는 여러개의 이력을 갖는다.
User table의 기본키를 키본키로서 상속받으며 한 유저 내에서 구분될 수 있는 resumeId를 기본키로 추가 한다.추가한다.
유저가 삭제되면 유저의 모든 이력이 삭제된다.

Attachment table
복합기본키 resumeId, userId, attatchmentId 
-설명-
이력에 첨부될 파일 주소를 담는다.
한개의 이력에는 여러개의 파일 주소가 담긴다.
Resume table의 기본키를 기본키로서 상속받으며 해당 이력내에서 구분될 수 있는 attchmentId를 기본키로 추가한다.
이력이 삭제되면 해당 이력에 포함된 모든 첨부파일을 삭제한다,

Board table
복합기본키 userId, boarId
- 설명 -
구인 게시글이며 구인 정보를 담는다. 
구인 게시글을 게시글로 축약해서 설명하겠다.
한명의 유저는 여러개의 게시글을 갖을 수 있다.
User table의 기본키를 기본키로서 상속받으며 한 유저내에서 게시글을 구분할 수 있는 boardId를 기본키로 추가한다.
유저가 삭제되면 게시글도 삭제된다.
게시물이 마감되면 BoardStore로 이전된 후 삭제된다.


Applier
복합기본키 userId, boardId 
-설명-
게시글에 지원한 구직 지원자들의 목록이다.
구직 지원자를 지원자로 축약해서 설명하겠다.
한 게시글은 여러명의 지원자를 가질 수 있다.
Board table의 boardId를 상속받는다. User tabled의 userId를 상속받는다.
이 상속받는 둘을 기본키로 하는 복합기본키로 한다.
게시글이 삭제되면 게시글의 참여한 모든 지원자는 삭제된다.
게시물이 마감되면 ApplierStore로 이전된 후 삭제된다.
// 지원되면 해당 게시판 주인에게 지원자가 있다는 알람을 보낸다.

BoardStore table( BoardHistory로 바꾸고자 한다)
복합기본키 userId, boardId
-설명-
마감된 게시글을 모아논 것이다.
Board table의 복합기본키를 그대로 상속받아 기본키로 사용한다.

ApplierStore table(ApplierHistory로 바꾸고자 한다)
복합기본키 userId, boardId
-설명-
마감된 게시글의 지원자들을 모아논 것이다.
Applier table의 복합기본키를 그대로 상속받아 기본키로 사용한다,

게시물 삭제 트리거,
-설명-
1. 게시글이 마감 요청을 받는다.
2. 해당 게시물을 BoardStore에 복제해서 저장한다.
3. 해당 게시물의 지원자가 존재하면 모든 지원자를 ApplierStore에 복제해서 저장한다
// 모든 지원자에게 게시물의 마감과 구인자에게 선택받은 결과를 알람으로 보낸다
3-1 해당 게시물의 지원자가 구인자에게 선택받으면 해당 지원자는 Resume table로 이동시키고 해당 게시물은 유저의 경력 테이블에 추가한다.(경력테이블 새로 설정)
3-2 해당 게시물을 삭제한다,
4. 해당 게시물의 지원자가 존재하지 않으면 해당 게시물을 삭제한다,
